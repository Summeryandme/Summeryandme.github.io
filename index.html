数据结构与算法-红黑树

发表于 2023-08-23

2-3 树

允许每一个节点拥有 1 或 2 个 key

  • 2-node：1 key，2 children
  • 3-node：2 key，3 children

具体节点的顺序如下图

查询某个节点很简单，就不赘述了，现在看如何做到插入，插入时在最底层寻找合适的位置与某个节点进行合并

  • 当插入的节点为2-node时，直接合并
  • 当插入的节点为3-node时
    • 暂时合并为一个4-node
    • 将4-node的中间节点移入其父节点中
    • 若父节点也变成了4-node，重复上面的步骤
    • 若根节点此时变成了4-node，将其拆分成 3 个2-node

2-3 tree 是完全平衡的，从根节点到叶子节点的每一条路径都有相同的长度
2-3 tree 直接实现起来比较复杂，因为维护多种类型的节点比较累，需要处理的 case 种类较多，于是有聪明的人们发明了红黑树

红黑树

  • 用一个二分搜索树代表了 2-3 树
  • 使用「内部」的「向左倾斜的」连接来作为3-node的连接

具体如下图所示

所以红黑树有以下几个特点

  • 每一个节点都不可能有两条红线与之相连
  • 每一条从根节点通往叶子节点的黑线数量是一样的
  • 红线只会向左倾斜

数据结构与算法-并查集

发表于 2023-08-22 更新于 2023-09-12

本文的行文思路和相关图片参考 Coursera: Algorithms I 课程

给定 N 个独立的对象，并查集主要解决以下两个问题：

  • 合并（union）：连接两个对象
  • 查询（find）：是否存在一条路径可以连通两个对象

对于上图的 union 过程，将对象 5 和对象 7 进行连接之后，1-7 均连通，若调用 find(1, 6)则应返回 true

接下来具体讨论并查集是如何实现的

Quick Find

给定 N 个独立的对象，有

  • int 类型的长度为 N 的数组id[]
  • 如果两个对象的 id 值相同，则他们是连通的

但是这种方法太慢了，在进行union操作时时间复杂度为O(N2)

Quick Union

相对于 Quick Find 不同的是，其数组代表的含义为：

  • id[i]为i的父亲节点
  • i的根节点为id[id[id[...id[i]...]]]

对于 Find 操作，只需要判断 p 和 q 是否具有相同的根节点
对于 Union 操作，只需将 p 的根节点设置为 q 的根节点

此种方法的缺点是，在最差的情况下，树会变得很高，导致进行 Find 操作时的开销很大

Improvement

Weighted quick-union

具体的数据结构和 Quick Union 一样，只是添加一个数组sz[i]来对i所在树的对象数量进行统计

  • Find 操作和 Quick Union 一致return root(p) == root(q);
  • Union 操作会把对象数量少的树连接到对象数量大的树

这样的优化点在于任何节点所在树的高度都不会超过lg N

path compression

思路非常简单，节点在查找自己的root时，最后将自己挂到root节点下，这样会使整棵树几乎是平的
